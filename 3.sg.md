# Security Group

인스턴스에 대해 인바운드 및 아웃바운드 트래픽을 제어하는 가상 방화벽 역할

→ VPC(하나의 서비스 전체를 나타내는 전체 영역)에서 인스턴스 시작할 때 최대 5개 보안 그룹에 인스턴스 할당 가능

보안 그룹은 서브넷 수준이 아니고, 인스턴스 수준에서 동작

→ VPC 내부 서브넷의 각 인스턴스는 서로 다른 보안 그룹에 할당 가능

인스턴스 시작 시, 특정 그룹 지정하지 않으면 자동으로 기본 보안 그룹 할당

# 특징

### 보안 장치

- Network Access List(NACL)과 함께 방화벽 역할을 하는 서비스

### Port 허용

- 트래픽이 지나갈 수 있는 Port, Source 설정 가능
- Deny 불가능 → NACL로 가능함

### 인스턴스 단위 적용

- 인스턴스 당 한 개 이상의 보안 그룹 설정 가능
- NACL의 경우 서브넷 단위
- 인스턴스는 설정된 모든 보안 그룹의 규칙을 따름

### 설정된 모든 룰을 이용한 필터링

- NACL의 경우 적용 룰의 순서대로 필터링

### Stateful

- 인바운드로 들어온 트래픽은 아웃바운드 설정 없이 나갈 수 있음
- NACL은 Stateless
